| Category | Examples | Severity Range |
|---|---|---|

| Data Exfiltration | Sending local data to external URLs | Critical–High |

## Frequently asked questions

**How do I scan MCP servers for security issues?**
Run `npx firmis scan --platform mcp` in your project directory. Firmis detects tool poisoning, data exfiltration, credential harvesting, and more. No configuration needed.

**What security risks exist in Claude Skills?**
Claude Skills can contain prompt injection attacks, hidden instructions that override agent behavior, hardcoded API keys, and unauthorized file system access. Firmis detects these with 45+ dedicated rules.

**Does Firmis upload my code?**
No. Firmis runs entirely offline. Your code never leaves your machine. No telemetry is collected by default.

**How do I add Firmis to my CI/CD pipeline?**
Run `npx firmis ci --fail-on high --format sarif` in your pipeline. This runs the full discover, BOM, scan, report pipeline and exits non-zero if high or critical findings are detected.

**What is an Agent Bill of Materials (Agent BOM)?**
An Agent BOM is a CycloneDX 1.7 inventory of all AI agent components, dependencies, and models in your project. Generate one with `npx firmis bom`.

**How do I detect prompt injection in AI agents?**
Firmis includes 13 prompt injection detection rules that scan tool descriptions, system prompts, and handler code for instruction override patterns, hidden directives, and context manipulation techniques.
